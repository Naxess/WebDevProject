CSS - Basics

CSS is not a programming language or a markup language. It is a style sheet language. It is used to selectively style HTML elements. A style sheet can be applied in 3 ways:

Description Example Usage
External CSS Refers to when the CSS stylesheet is on a separate file from the HTML page. The HTML page must reference this external CSS stylesheet file by using the link tag. The rel attribute must be set to stylesheet and the href attribute must be set to the location of the file. The link tag should be defined inside of the head section of the HTML page.
HTML Code:
<html>
    <head>
        <link rel="stylesheet" href="external_css_file.css">
    </head>
    ...
</html>
external_css_file.css:
body{}
    background-color:ghostwhite;}
    margin:15px;}
}}
h1{}
    color:red;}
}
Internal CSS When CSS style sheets are defined within the HTML page, it is referred to as internal CSS. To do this, create a style tag inside of the head section of the page. Finally, supply the CSS properties/values inside the style tag. There are no attributes that need to be specified. 
<html>
    <head>
        <style>
            body{
                background-color:ghostwhite;
                margin:15px;
            }
            h1{
                color:red;
            }
        </style>
    </head>
    ...
</html>
Inline CSS Used to apply CSS styling to a single element. This is done by adding the style attribute to the element, then supplying that attribute with the desired CSS properties/values. 
<html>
    ...
    <body style="background-color:ghostwhite;margin:15px;">
        <h1 style="color:red;">Hello</h1>
    </body>
</html>

Box Model

The CSS Box Model applies to block boxes and defines how different parts of a box - margin, border, padding, and content - work together to create a box that you can see on a page.
Additionally, there are multiple types of boxes: content box, padding box, border box, and margin box.

  • Padding: refers to the space directly around the outside of the content.
  • Border: refers to the visible line that appears directly around the outside of the padding.
  • Margin: refers to the space directly around the outside of the border.
  • Content box: refers to the area where the content is displayed. It can be sized using properties like: inline-size, block-size, width, and height
  • Padding box: refers to the space that sits around the content as whitespace. It can be sized using padding and related properties.
  • Border box: this box wraps the content and any padding. It can be sized using border and related properties.
  • Margin box: this box is the outermost layer, wrapping the content, padding, and border as whitespace between this box and other elements. It can be sized using margin and related properties.
Example usage:
.box {
    width: 350px;
    height: 150px;
    margin: 10px;
    padding: 25px;
    border: 5px;
}

Alternate Box Model

In the alternative box model, any width is the width of the visible box on the page. The content area width is that width minus the width for the padding and border. Because of this, there's no need to add up the border and padding to get the real size of the box.
To enable the alternate box model, we add the box-sizing property and set the value to content-box or border-box, depending on the desired effect.

  • content-box: gives the default CSS box-sizing behavior. If an element's width is set to 100 pixels, then the element's content will be 100 pixels wide and the width of any border or padding will be added to the final rendered width, making the element wider than 100px;
  • border-box: tells the browser to account for any border and padding (but not the margin) in the values specified for an element's width and height. If an element's width is set to 100 pixels, that 100 pixels will include any border or padding that was added, and the content box will shrink to absorb that extra width. This typically makes it easier to size elements.
Example usage:
.alt_box {
    box-sizing: border-box; # Other option: content-box
}

Backgrounds & Borders

Backgrounds
Property Description
background-color Defines the background color on any element in CSS. It accepts any valid <color>:
  • Keyword syntax: uses a <named-color> (e.g. red, blue, green, orange, yellow, transparent, etc..)
  • Hexadecimal notation syntax: uses <hex-color> (e.g. #ff0000, #00ff00, #0000ff, #ffffff, #000000, etc..)
  • Functional notations syntax: (e.g. rgb(255,0,0), rgb(0,255,0), rgba(0,0,255, 0.5), rgba(0,0,255 / 50%), etc..)
  • Mixing colors syntax: (e.g. color-mix(in lch, red, white), color-mix(in lch, red 50%, white), color-mix(in srgb, #ff0000 50%, white))
background-image Enables the display of an image in the background of an element. 
// Example usages:
background-image:url("../media/profile_pic.png"); // image can be internal or from an online source

// Can be supplied with multiple images, delimited by a comma
// If multiple images are supplied, they will overlap, with higher priority given in the order in which they were supplied
background-image:url("../arlan.webp"), url("../asta.webp");

// Creates a background using the colors supplied
// The color will gradate from the first supplied color to the other, in the direction supplied
// (Other directions: 'to top', 'to bottom', 'to left');
background-image:linear-gradient(to right, rgba(0,0,255,0.5), rgba(255,0,0,0.5)), url("../argenti.webp");
background-repeat This property is used to control the tiling behavior of images. Available values: no-repeat, repeat-x, repeat-y, and repeat 
// Example usage:
.box {
    background-image:url("profile_pic.png");
    background-repeat: repeat;
}
background-size This property takes a <length>, <percentage> value or the cover or contain keywords to size the image to fit inside the background.
background-position This property allows the position of a background image to be edited in relation to the box to it is applied to. It takes many values in several forms (coordinate, percentage, length, etc..).
  • Coordinate syntax takes two values, an x value and a y value, which correspond to the image's x and y axis.
    • Example usage: background-position: 50px 100px;
  • Keyword syntax takes one or two values. Acceptable values: top, bottom, left, right, center. If using two values, the directions should not contradict each other as they will cancel out.
    • Example usage: background-position: top right;
  • Percentage syntax values.
    • Example usage: background-position: 25% 75%;
  • Length syntax values.
    • Example usage: background-position: 0 0;
  • Other combinations exist as well. See reference.
background-attachment This property allows specifying how background images scroll when the content scrolls. It accepts the following values:
  • scroll: causes the element's background to scroll when the page is scrolled. If the element content is scrolled, the background does not move. In effect, the background is fixed to the same position on the page, so it scrolls as the page scrolls.
  • fixed: causes an element's background to be fixed to the viewport so that it doesn't scroll when the page or element content is scrolled. It will alwaysremain in the same position on the screen.
  • local: fixes the background to the element it is set on, so when the element is scrolled, the background scrolls with it.

Borders

Property that draws a border around the element. Similar to the outline property. Differences being that outlines never take up space (as theyr are drawn outside of an element's content) and they don't have to be rectangular.

Property Description
border Draws a border around the element using border-width, border-style, and border-color values. The property may be specified using one, two, or three of the values. The order in which they're supplied does not matter.
  • border-width/line-width: sets the thickness of the border. Defaults to medium if absent.
    Subcategories: border-top-width, border-bottom-width, border-left-width, border-right-width
    Example usage: 
    border-width:5px; // Sets the width of all sides of the border to 5px;
border-top-width:10px; // Sets the width of only the top width to 10px;
  • border-style/line-style: sets the style of the border. Defaults to none if absent.
    Subcategories: border-top-style, border-bottom-style, border-left-style, border-right-style
  • border-color/color: sets the color of the border. Defaults to currentcolor if absent.
    Subcategories: border-top-color, border-bottom-color, border-left-color, border-right-color
border-radius This property is used to round the corners on borders. This property accepts up to two lengths or percentages, which may be used as a value. The first value defines the horizontal radius, the second defines the vertical radius.
Subcategories: border-top-right-radius, border-top-left-radius, border-bottom-left-radius, border-bottom-right-radius

Fonts and Text

The appearance of text can be modified using CSS properties. Things like the font size, color, placement, styling, etc.. are all appearances that can be edited in CSS. Additionally, we can supply fonts from other sources if desired.

Font Properties
  • color: sets the color of the foreground content of the selected elements, usually the text, but can also include other things like underlines, overlines, etc..
    Acceptable values: named-colors (red, blue, green, etc.., rgb/rgba (rgb(255 255 0), rgba(0 255 255 0.5), etc..), and hexidecimal values (#ffffff, #000000, etc..).
  • font-family: applies a specified font(s) to selected elements. Some web-safe (i.e.common across a wide variety of browsers) fonts: Arial, Courier New, Georgia, Times New Roman, Trebuchet MS, Verdana. Default fonts: serif, sans-serif, monospace, cursive,fantasy.

    Multiple fonts can be supplied as backups in case specified fonts are unavailable. Example: font-family: "Trebuchet MS", Verdana, sans-serif. If 'Trebuchet MS' is unavailable, then 'Verdana' will be tried, if 'Verdana' unavailable, then sans-serif, etc.. This is called font stacking.
  • font-size: resizes element text to the specified size. Takes values in the form of px, em, and rem.
    • px (pixels): refers to the number of pixels. Sets the text height to the number of pixels specified.
    • em: refers to the font size set on the parent element.
    • rem: refers to the font size set on the root element of the document.
  • font-style: used to turn italic text on or off. Possible values: normal, italic, and oblique.
  • font-weight: used to set boldness. Possible values: light, normal, bold, black, and a value between 100-900).
  • text-transform: enables/disables text transformation. Possible values: none (prevents transformation), uppercase, lowercase, capitalize, full-width (transforms all glyphs to be written inside a fixed-width square, similar to a monospace font, allowing aligning of e.g., Latin characters along with Asian language glyphs (like Chinese, Japanese, Korean).
  • text-decoration: sets/unsets text decorations on fonts. Possible values: none, underline, overline, and line-through.
  • text-shadow: applies shadows to text. Accepts two to four values. These four properties in the order from left to right:
    • The horizontal offset of the shadow from the original text. Can take px values. Required.
    • The vertical offset of the shadow from the original text. Can take px values. Required.
    • The blur radius - a higher value means the shadow is dispersed more widely. Defaults to 0 if not specified.
    • The base color of the shadow, which can take any CSS color unit. If not included, it defaults to currentcolor.

    Multiple shadows can be applied by adding additional rows of values, delimited by commas. Example: 
        h1 {
        text-shadow:
            1px 1px 2px red,
            -1px -1px 2px blue,
    }
  • text-align: this property is used to control how text is aligned within its containing content box. Possible values:
    • left: left-justifies the text
    • right: right-justifies the text
    • center: centers the text
    • justify: spreads the text out, varying the gaps between the words so that all lines of text are the same width.
  • line-height: this property sets the height of each line of text. Takes a numeric value. Example: line-height: 1.5;
  • letter-spacing: this property sets the spacing between letters in the text
  • word-spacing: this property sets the spacing between words in the text
  • Additional font styles and text layouts

Text Directionality

The writing-mode property allows switching from one writing mode to another. Writing modes in CSS refer to whether the text is running horizontally or vertically.
It accepts three possible values:

  • horizontal-tb: sets the flow direction to top-to-bottom. Sentences run horizontally.
  • vertical-rl: sets the flow direction to right-to-left. Sentences run vertically.
  • vertical-lr: sets the flow direction to left-to-right. Sentences run vertically.

Essentially it is setting the direction in which block-level elements are displayed on the page - either in top-to-bottom, right-to-left, or left-to-right. This then dictates the direction that the text flows in sentences.

Example usages:
.horizontal {
    writing-mode: horizontal-tb;
}

.vertical {
    writing-mode: vertical-rl;
}

Overflowing Content

If the text in the content starts to overflow, then the inline-size property can be used to adjust the height/width of the element. If the the writing mode is vertically oriented, then the inline-size property will relate to the height of the element. If the writing mode is horizontally oriented, then the inline-size property will relate to the width of the element.

Example usage:
.box {
    writing-mode: vertical-lr;
    inline-size: 300px;
}

Using min- and max-

Using the min- and max- properties in combination with properties like height and width allows us to set a minimum or maximum size for that property.

min-
min-height Sets the minimum allowed height of an element.
min-width Sets the minimum allowed width of an element.
min-inline-size Defines the minimum horizontal or vertical size of an element's block, depending on its writing mode. It correspondes to either the min-width or min-height property, depending on the value of writing-mode. If the writing mode is vertically oriented, the value of min-inline-size relates to the minimum height of the element; otherwise it relates to the minimum width of the element. A related property is min-block-size, which defines the other dimension of the element.
min-block-size Sets the minimum horizontal or vertical size of an element's block depending on the writing direction specified by writing-mode. If the writing mode is vertically oriented, the value of min-block-size relates to the minimum width of the element; otherwise, it relates to the minimum height of the element. min-inline-size defines the other dimension of the element.
max-
max-height Sets the maximum allowed height of an element.
max-width Sets the maximum allowed width of an element.
max-inline-size Defines the maximum horizontal or vertical size of an element's block depending on its writing mode. It corresponds to either the max-width or the max-height property, depending on the value of the writing-mode. If the writing mode is vertically oriented, the value of max-inline-size relates to the maximum height of the element; otherwise, it relates to the maximum width of the element. A related property is max-block-size, which defines the other dimension of the element.
max-block-size Sets the maximum size of an element in the direction opposite that of the writing direction specified by writing-mode. i.e. if the writing direction is horizontal, then max-block-size is equivalent to max-height; if the writing direction is vertical, then max-block-size is the same as max-width. The other dimension's maximum length is specified using the max-inline-size.

Viewport Units

The viewport is the visible area of the webpage in the browser. vw is the viewport width, while vh is the viewport height. Using these units, elements can be sized relative to the viewport of the user. 1vh is equal to 1% of the viewport height, and 1wh is equal to 1% of the viewport width. These are applicable to boxes and text as well.

Images, Media, and Form Elements

The object-fit property

For media, such as <img>, <video>, and <iframe> elements, the object-fit property can be used to size the element to fit a box in a variety of ways.
Example usage: 

img,video,iframe {
    object-fit:fill;
}
object-fit values:
Values Description
contain The replaced content is scaled to maintain its aspect ratio while fitting within the element's content box. The entire object is made to fill the box, while preserving its aspect ratio, so the object will be letterboxed if its aspect ratio doesn't match the aspect ratio of the box.
cover The replaced content is sized to maintain its aspect ratio while filling the element's entire content box. If the object's aspect ratio doesn't match the aspect ratio of its box, then the object will be clipped to fit.
fill The replaced content is sized to fill the element's content box. The entire object will completely fill the box. If the object's aspect ratio does not match the aspect ratio of its box, then the object will be stretched to fit.
none The replaced content is not resized.
scale-down The content is sized as if none or contain were specified, whichever would result in a smaller concrete object size.

The overflow property

The overflow property can be used to address situations in which the content does not fit within the parent element box in the horizontal and/or vertical direction.

Subcategories of the overflow property are overflow-x (for controlling behavior at left and right edges) and overflow-y (for controlling behavior at the top and bottom edges).

This property can accept one or two values. If only one value is supplied, then it applies to both overflow-x and overflow-y. If two values are supplied,then the first value applies to overflow-x and the second value applies to overflow-y.
Example usage: 

img,video,iframe {
    overflow:hidden;
}
overflow values:
Values Description
visible Overflow content is not clipped and may be visible outside the element's padding box. The element box is not a scroll container. This is the property's default value.
hidden Overflow content is clipped at the element's padding boxing. There are no scroll bars, and the clipped content is not visible, but still exists. User agents do not add scroll bars and also do not allow users to view the content outside the clipped region by actions such as dragging or scrolling. It can, however, be scrolled programmatically by setting the value of the scrollLeft property or scrollTo() method, in which case the element box is a scroll container.
clip Overflow content is clipped at the element's overflow clip edge that is defined using the overflow-clip-margin property. As a result, content overflows the element's padding box by the <length> value of overflow-clip-margin or by 0px if not set. Overflow content outside the clipped region is not visible, user agents do not add a scroll bar, and programmatic scrolling is also not supported. No new formatting context is created.
scroll Overflow content is clipped at the element's padding box, and overflow content can be scrolled into view using scroll bars. User agents display scroll bars in both horizontal and vertical directions if only one value is set, whether or not any content is overflowing or clipped. The use of this keyword can prevent scroll bars from appearing and disappearing as content changes. The element box is a scroll container.
auto Overflow content is clipped at the element's padding box, and overflow content can be scrolled into view. Unlike scroll, user agents display scroll bars ONLY if the content is overflowing and hides scroll bars by default. If content fits inside the element's padding boxing, it looks the same as with visible but still establishes a new formatting context. The element box is a scroll container.

Other Table CSS Properties
Property Description
table-layout Set the algorithm that is used to lay out table cells, rows, and columns. There are two values: auto and fixed.
  • auto: enables an algorithm that automatically adjusts the widths of the table and its cells to accomodate the content. This is enabled by default.
  • fixed: enables the fixed table layout algorithm. When enabled, the table's width NEEDS to be specified explicitly using the width property. If the width is not specified, then, then the browser uses the automatic table layout algorithm, in which case the fixed value has no effect.
    • The advantage of the fixed algorithm is that it is faster than auto on account of not needing to calculate the column width, borders or cell spacing. These widths can be specified explicitly. If not, then the widths are divided equally with no regard to the content.
    • The downside is that it doesn't take the content into account, meaning there may be overlap or poor appearance.
border-collapse This property sets whether cells inside a table have shared or separate borders. It takes one of two values: collapse or separate.
  • collapse: adjacent cells have shared borders
  • separate: adjacent cells have distinct borders

Debugging CSS

In a browser, we can view access the DevTool by pressing F12 or by right-clicking and selecting 'Inspect'. This allows us to view the webpage's source. The source displays the html and css code of the webpage viewable under the 'Elements' and 'Styles' tabs, respectively. It also allows us to edit both, displaying the changes in real-time, thus making it a valuable tool for testing changes.

Organizing CSS
Tips for keeping the CSS code organized:
  • Utilize a coding style guide.
  • Consistency in things such as naming, color description, formatting, etc..
  • Make the code readable through good formatting.
  • Use comments to describe or explain potentially confusing code. 
        /* Commenting in CSS is done like this */
    /* This comment
    is broken into
    multiple lines */
  • Create logical sections. e.g. a section for general styles, another for utilities, global styles/utilities, etc..
  • Avoidance of overly-specific selectors.
  • Break large stylesheets into multiple smaller ones. Consider separate stylesheets for different sections of the website and another for global styles. (This is attainable by linking to multiple stylesheets from the same page.)
