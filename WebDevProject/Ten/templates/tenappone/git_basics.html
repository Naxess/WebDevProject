Git - Basics

Git is a distributed version-control system designed to track changes in any set of computer files, usually used for coordinating work among programmers who are collaboratively developing source code during software development.

Version Control

Refers to the practice of tracking and managing changes to software code. Version control systems are software tools that help software teams manage changes to the source code over tme. Every single modification to the code is tracked in a special database, thus allowing developers to compare their code to earlier versions and if necessary, revert to them.

Benefits:

  • A history of every change to every file is tracked and stored for later viewing or as a backup to revert to.
  • Branching and merging. Allows multiple people to work on the same code simultaneously. Users can create local copies of the code on which they can make modifications that they can then merge into the original code.
  • Traceability and documentation. Can be used with project management and bug tracking software such as Jira. Users are also able to submit messages with their changes. These messages can include useful details about the any new features/fixes that are being implemented with their change.

Benefits of using Git

Git, at the moment of this writing, is the most widely use version control system in the world.

Benefits of using Git:

  • Better performance
  • High security
  • Flexibility. Supports various kinds of nonlinear development workflows. High efficiency in both small and large projects. Wide compatibility with existing systems and protocols

Git/GitHub Installation/Setup

Before we can start using Git and GitHub, we need to install them first:

  • Install Git from https://git-scm.com/download/win. During installation, make sure to enable the option to add Git to the PATH.
  • Create a GitHub account on https://github.com/.
  • Verify whether Git was installed correctly (or verify version), open the Command Prompt and enter the following command: git --version
    The should look something like this: git version 2.44.0.windows.1
    If, instead, you get an error, then try rebooting the PC. Failing that, try reinstalling and ensuring that the option to add Git to the PATH is installed.

Git vs GitHub
  • The difference between Git and GitHub is that Git is a version control software used to keep track of the changes in your code.
  • GitHub is a company and website that assists with managing Git and hosting the files on their website. They also ofer extra tools for collaboration.

Setting up a Git repository - using git init
  1. In the GitHub website, create a repository. Locate the HTTPS url for the repo. If SSH is selected, then click on HTTPS to display the url instead. Save this url for later use.
  2. On your PC, create a directory that you want to designate as a Git repo. This will be where your projects will be stored. Copy the path of this directory.
  3. Open the Command Prompt application and use commands to navigate into the directory you created in the last step. Enter the following command to designate this directory as a Git repo: git init
    (This installs some files onto the directory. Do not delete or edit them.)

Adding local projects to the (remote) Git repository
  1. After the local Git repo is set up, add projects by copy-pasting them into the root directory of the Git repo.

  2. Open the Command Prompt and navigate to the root directory of the Git repository. Run the following command on the added project(s): git add .

    This command adds new or edited files into a list of files (staging area) to be added/updated into the remote repo.

    The period ( . ) added at the end is a wildcard used to select every project in the directory. If we added multiple projects, then all will be added. If, instead, we want to select a single project, use this format instead: git add project_name

  3. The following command takes a snapshot of the new/edited files in the staging area: git commit -m "message"

    Each commit requires that a message be attached, describing the list of changes that will occur to the remote repo once the file has been pushed. Example: git commit -m "Adding a new project to the repo."

  4. Now we need to tell Git which remote repo that we want to send the new changes to. Copy the HTTPS url from the Git repo on the GitHub website and run the following command on that url: git remote add origin https_url

    Once we have designated the Git repo as the destination, all future changes will be pushed to this location.

  5. The last step to propagate the changes to the remote repo requires running this command: git push -u origin master

    After the command is run, all of the changes in the commit(s) are then applied onto the remote repo. We can then view the added/edited projects/files in GitHub.

    (You may be prompted to sign in after running this command.)

Copying a remote repo to your local machine - using git clone

We can make a copy of a remote repository onto our local machine by cloning it - we simply need the HTTPS url of this remote repository. Once we have the url, we can run the following command to make a local copy of the repo: git clone https_url

Applying changes from the local to the remote repo - using git push

The git push command is used to update the remote repo with your local changes. It takes the commits you created and applies those changes to the remote repo.
