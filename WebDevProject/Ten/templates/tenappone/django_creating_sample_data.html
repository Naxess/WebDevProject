Django - Creating/Editing Sample Data/Rows

Example of Viewing/Adding entries via Python:

 
from example_app.models import Albums

albums_entries = Albums.objects.all()               # Retrieves all entries in the Albums table into a QuerySet object
print(albums_entries)                               # Output should be an empty QuerySet

new_row = Albums(artist="John Doe", tracks=5)       # Creates a row with populated column. (NOT saved into the table yet!)

print(new_row.artist)                               # Returns the value of the row's artist column
print(new_row.tracks)                               # Returns the value of the row's tracks column

new_row.tracks = 12                                 # Updates the value of the tracks column

new_row.save()                                      # Saves the row into the database

print(Albums.objects.all())                         # The output should have a single entry: QuerySet[<Album: Album object(1)>]

Viewing/Adding Entries via the Faker Module (Basic Guide)

Installing the Faker Module

The Faker library can be installed by running the following command in the command prompt: 

pip install Faker

Importing and Setting Up the Faker Module

The first step to using the library in your Py script is to import it using the following command: 

from faker import Faker

The second step is to create a "generator" with this command:

faker_generator = Faker()

Generating Data from the Faker Module

After creating the generator, we can use its built-in methods to generate various types of data, such as names, first names, last names, addresses, urls, emails, company names, etc.. Example usage: 

from faker import Faker
faker_generator = Faker()

fake_name = faker_generator.name()              # Randomly selects a name
fake_url = faker_generator.url()                # Randomly generates a url
fake_date = faker_generator.date()              # Randomly generates a date
fake_company = faker_generator.company()        # Randomly generates a company

Using Faker with Django

Python Script Setup

Before a Py script can start manipulating models, it needs to import the os and django modules. Then it needs to set an environmental variable using the imported os module. The variable name ('DJANGO_SETTINGS_MODULE') should be pointed to the project's settings file (formatted like so: 'project_name.settings'). The last step is to call the django module's setup method, like so: django.setup().

Example: 

import os
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'first_project.settings')
import django
django.setup()

After the initial setup is complete, the next step is to import the models from the app. Example: 

from first_app.models import Album, Artist, Song

Retrieving All Entries:

Example: 

Album.objects.all()

Creating and Saving a New Entry:

t = Album.objects.get_or_create(title="SuperBeats", artist="John Bam", songs="Sad|Happy|Melancholy")[0]
t.save() # Saves the entry to the database
