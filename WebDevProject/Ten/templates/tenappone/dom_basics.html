QuerySelector

Selects the first element that matches the search criteria.

const e1 = document.querySelector('div');                   // Selects the very first 'div' element
const e2 = document.querySelector('#abc');                  // Selects the element with 'id' attribute value of "abc"
const e3 = document.querySelector('.jkl');                  // Selects the first element with 'class' attribute value of "jkl"

QuerySelectorAll

Selects all elements that match the search criteria.

const e4 = document.querySelectorAll('div');                // Selects all div elements
const e5 = document.querySelectorAll('li a');               // Selects all 'a' elements inside of 'li' elements

GetElementById

Selects the element whose id attribute matches the search criteria.

const e6 = document.getElementById('abc');

GetElementsByClassName

Selects all elements whose class attribute matches the search criteria

const e7 = document.getElementsByClassName('jkl');

GetElementsByName

Selects all elements whose name attribute matches the search criteria.

const e8 = document.getElementsByName('ghi');

GetElementsByTagName

Selects all elements whose tag name (element name) matches the search criteria.

const e9 = document.getElementsByTagName('a');              // Retrieves all 'a' elements
const e10 = document.getElementsByTagName('h1');            // Retrieves all 'h1' elements
const e11 = document.getElementsByTagName('*');             // Retrieves all elements

GetElementsByTagNameNS(namespace, tagname)

Selects all elements whose namespace and tag name match the search criteria. The namespace is the value attached to the xmlns attribute. 

const e12 = document.getElementsByTagNameNS('http://www.w3.org/1999/xhtml', 'p');

Events

Types of events that can be listened to using the addEventListener() method or using the oneventname property of this interface. Example:
let headerOne = document.querySelector('h1');
headerOne.addEventListener(<eventName>, exampleFunction);

Keyboard Events
keydown Fired when a key is pressed
keypress Fired when a key that produces a character value is pressed
keyup Fired when a key is released

Mouse Events
click Fired when the left mouse button is pressed and released on the element
dblclick Fired when the left mouse button double-clicks on the element
auxclick Fired when any mouse button other than the left button has been pressed and released on the element
mousedown Fired when the left mouse button is pressed on the element
mouseenter Fired when the pointer is moved over the element
mouseleave Fired when the pointer is moved out of the element
mousemove Fired when the pointer is moved while over an element
mouseup Fired when the left mouse button is released over an element
mousewheel Fired when the mouse wheel or similar device is operated (scrolled)
contextmenu Fired when the user attempts to open a context menu
DOMActivate Occurs when the element is activated, for instance, through a mouse click or keypress
DOMMouseScroll Occurs when a mouse wheel or similar device is operated and the accumulated scroll amount is over 1 line or 1 page since the last event
MozMousePixelScroll Fired when the mouse wheel or similar device is operated
webkitmouseforcechanged Fired each time the amount of pressure changes on the trackpad touchscreen
webkitmouseforcedown Fired after the mousedown event as soon as sufficient pressure has been applied (force click)
webkitmouseforcewillbegin Fired before the mousedown event
webkitmouseforceup Fired after the webkitmouseforcedown event as soon as the pressure has been reduced enough to end the force click

Pointer Events
getpointercapture Fired when an element captures a pointer using setPointerCapture()
lostpointercapture Fired when a captured pointer event is released
pointercancel Fired when a pointer event is canceled
pointerdown Fired when a pointer becomes active
pointercenter Fired when a pointer is moved into the hit test boundaries of an element or its descendants
pointerleave Fired when a pointer is moved out of the hit test boundaries of an element
pointermove Fired when a pointer changes coordinates
pointerout Fired when a pointer is moved out of the hit test boundaries
pointerover Fired when a pointer is moved into an element's hit test boundaries
pointerrawupdate Fired when a pointer changes any properties that don't fire pointerdown or pointerup events
pointerup Fired when a pointer is no longer active

Pointer Lock Events
pointerlockchange Fired when the pointer is locked/unlocked
pointerlockerror Fired when locking the pointer failed

Clipboard Events
copy Fired when the user initiates a copy action through the browser's UI
cut Fired when the user initiates a cut action through the browser's UI
paste Fired when the user initiates a paste action through the browser's UI

Scroll Events
scroll Fired when the document view or an element has been scrolled
scrollend Fired when the document view or an element has stopped scrolling
wheel Fired when the user rotates a wheel button on a pointing device (typically a mouse)

Fullscreen Events
fullscreenchange Fired when the Document transition into or out of fullscreen mode
fullscreenerror Fired if an error occurs while attempting to switch into/out of fullscreen mode

Animation Events
animationcancel Fired when an animation unexpectedly aborts
animationend Fired when an animation has completed normally
animationiteration Fired when an animation iteration has completed
animationstart Fired when an animation starts

Focus Events
blur Fired when an element has lost focus
focus Fired when an element has gained focus
focusin Fired when an element has gained focus, after focus
focusout Fired when an element has lost after, after blur

Load & Unload Events
DOMContentLoaded Fired when the document has been completely loaded and parsed without waiting for stylesheets, images, and subframes to finish loading
readystatechange Fired when the readyState attribute of a document has changed

Selection Events
selectionchange Fired when the current text selection on a document is changed

Composition Events
compositionend Fired when a text composition system such as an input method editor completes/cancels the current session
compositionstart Fired when a text composition system such as an input method editor starts a new session
compositionupdated Fired when a new character is received in the context of a text composition session

Touch Events
gesturechange Fired when digits move during a touch gesture
gestureend Fired when there are no longer multiple fingers contacting the touch surface, ending the gesture
gesturestart Fired when multiple fingers contact the touch surface, starting a new gesture
touchcancel Fired when one or more touch points have been disrupted in an implementation-specific manner
touchend Fired when one or more touch points are removed from the touch surface
touchmove Fired when one or more touch points are moved along the touch surface
touchstart Fired when one or more touch points are placed on the touch surface

Transition Events
transitioncancel An Event fired when a CSS transition has been cancelled
transitionend An Event fired when a CSS transition has finished playing
transitionrun An Event fired when a CSS transition is created
transitionstart An Event fired when a CSS transition has started transitioning
prerenderingchange Fired on a prerendered document when it is activated (i.e. user views the page)
securitypolicyviolation Fired when a content security policy is violated
visibilitychange Fired when the content of a tab has been visible or has been hidden

Miscellaneous Events
afterscriptexecute Fired when a static script element finishes executing
beforescriptexecute Fired when a static script is about to start executing
beforematch Fired on an element that is in the hidden until found state
contentvisibilityautostatechange Fires on any element with 'content-visibility: auto' set on it when it starts/stops being relevant to the user and skipping its contents
prerenderingchange Fired on a prerendered document when it is activated (i.e. user views the page)
securitypolicyviolation Fired when a content security policy is violated
visibilitychange Fired when the content of a tab has been visible or has been hidden

addEventListener() method

The addEventListener() method of the EventTarget interface sets up a function that will be called whenever the specified event is delivered to the target.

Syntax:
let elementName = document.querySelector('cssSelectors');
elementName.addEventListener('eventType', exampleFunction/listener);
elementName.addEventListener('eventType', exampleFunction/listener, options);
elementName.addEventListener('eventType', exampleFunction/listener, useCapture);
Example usages:
let first_header_one = document.querySelector('h1');

first_header_one.addEventListener('click', function(){first_header_one.style.color="blue";}
first_header_one.addEventListener('click', function(){first_header_one.style.color="blue";}

createElement() and createTextNode() methods
Example usages:
const heading = document.createElement('h1');                   // Creates a header element
const headingText = document.createTextNode('Hello World');     // Creates a text node that we can add text to

heading.appendChild(headingText);                               // Adds the text node to the header element
document.body.appendChild(heading);                             // Adds the header element into the body

function getRandomColor(){
    var numbers_and_letters = "0123456789ABCDEF";
    var color = "#";
    for(var i = 0; i < 6; i++){
        color += numbers_and_letters[Math.floor(Math.random()*16)];
    }
    return color;
};
