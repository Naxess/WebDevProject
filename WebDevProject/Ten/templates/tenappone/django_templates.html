Django - Templates

Templates contain the static parts of an HTML page - these are the parts that are always the same, no matter which page is opened. It is a convenient way to generate HTML dynamically.

Template Initial Setup:

  • Before we can start using templates, we need to create a directory to store them. This directory needs to be created at the root directory of the project and named 'templates'. For each app in the project, we need to create a subdirectory, named after each app. For example, if we have an app named 'first_app', then we need to create a folder named 'first_app' in the 'templates' directory.

  • The next step is to register the templates directory. Open the settings.py file, and run the following steps:

    • Import the os module (if it hasn't been imported already). (Alternatively, can import just the path function from the os module. If you do this, adjust the value in the next step accordingly.) 

      import os

    • Next, locate a variable named BASE_DIR. Directly below this variable, create another variable named TEMPLATE_DIR. Set the value equal to os.path.join(BASE_DIR, 'templates'). 

      BASE_DIR = Path(__file__).resolve().parent.parent
TEMPLATE_DIR = os.path.join(BASE_DIR, 'templates')
      (Optional) Breakdown of BASE_DIR:
      __file__ Outputs the path and name of the current file
      Path(__file__) Creates a Path object, on which built-in methods can be run
      Path(__file__).resolve() Returns the Path object with all symlinks removed
      Path(__file__).resolve().parent.parent The parent function returns the directory above the current one. It is run twice in this example, so it returns a directory from two levels above the current one.
    • Next, locate the TEMPLATES list object. There should be a dictionary inside, which should contain a key named "DIRS" with the value being a list object. Add TEMPLATE_DIR to it. 
      TEMPLATES = [
    {
        "BACKEND": "django.template.backends.django.DjangoTemplates",
        "DIRS": [TEMPLATE_DIR, ],
...

Creating a template and mapping it:
  • When creating a template for an app, then it should be created within that app's directory, otherwise the root directory of the templates folder is fine. With that determined, we can create the HTML files that we'll use as our templates. We'll also make the necessary changes in mapping to accommodate the template.

    • Example template:
      [.../first_project/templates/first_app/index.html] 

      <!DOCTYPE html>
<html>
    <head>
        <title>First App</title>
    </head>
    <body>
        <h1>Hello, this is the index.html page!</h1>
    </body>
</html>

    • Example view function:
      [.../first_project/first_app/views.py] 

      ...
def index(request):
   return render(request, 'first_app/index.html')
...

    • Example path:
      [.../first_project/first_app/urls.py] 

      urlpatterns = [
    path('home/', views.index),
]

Template Variables

Template variables are a feature that renders data from the view onto the HTML page. The data sent to the template from the view is formatted as a dictionary (list of key-value pairs; aka map in other programming languages). One way to render the data is to select a key associated with the desired data, add it to the template in the desired location and enclose it in double brackets, like so: {{"{"}}{{"{"}} key {{"}"}}{{"}"}}.

Template Tag Lookups

Dictionary lookup {{"{"}}{{"{"}} key {{"}"}}{{"}"}}
Dictionary lookup {{"{"}}{{"{"}} example_dict.key {{"}"}}{{"}"}}
Attribute lookup {{"{"}}{{"{"}} example_object.attribute {{"}"}}{{"}"}}
List-index lookup {{"{"}}{{"{"}} example_list.0 {{"}"}}{{"}"}}
Send data from the view to the template:
  • In views.py, we need to modify the view function associated with the template.

    • In the body of the function create a dictionary, then populate it with key-value pairs.
      Example: 

      def index(request):
    example_dict = {'first_name':'John', 'last_name':'Doe', 'scores':[100, 90, 95, 100, 92]}
    return render(request, 'first_app/index.html')
    • In the return statement, the render function needs to be modified to include another argument: context. Set the value of context to the dictionary object that was just created.
      Example: 
      def index(request):
    example_dict = {'first_name':'John', 'last_name':'Doe', 'scores':[100, 90, 95, 100, 92]}
    return render(request, 'first_app/index.html', context=example_dict)
      With that done, every time the view function is called, it sends the dictionary to the template.
  • On the HTML page, render the desired value(s) by writing the associated key (or other lookup type) to the desired place(s), then enclosing that key in double brackets, like this: {{"{"}}{{"{"}} key {{"}"}}{{"}"}}.
    Example: 
    <!DOCTYPE html>
<html>
    <head>
        <title>First App</title>
    </head>
    <body>
        <h1>Hello, this is the index.html page!</h1>
        <p>Name: {{"{"}}{{"{"}} last_name {{"}"}}{{"}"}}, {{"{"}}{{"{"}} first_name {{"}"}}{{"}"}}</p>
    </body>
</html>
    When the above example webpage is loaded in a browser, you should see the below:

    Hello, this is the index.html page!

    Name: Doe, John

Template Tags

Template tags allow the use of programming logic on the data from the view. Tags are identified by the way they are enclosed with single-bracket, single percentage signs, like so:
{% verbatim %}{% if first_name == "John" %} ... {% endif %}{% endverbatim %}.
Examples of logic: loops, if-else, with, include, etc..

lorem

Displays random 'lorem ipsum' Latin text. Used to generate sample filler text. It can 3 arguments: count, method, and random.

count: optional, specifies the number of words or paragraphs to generate

method: optional, specifies content type: w for words, p for paragraphs, bfor plain-text paragraph blocks.

random: optional, if provided, then does not use the 'lorem ipsum' for filler text. 

{{"{"}}% lorem count method random %{{"}"}}

{{"{"}}% lorem %{{"}"}}
{{"{"}}% lorem 10 b %{{"}"}}
{{"{"}}% lorem 2 p %{{"}"}}

Built-in Template Filters

Django's Template Engine provides filters that are used to transform or filter the values or variables and tag arguments. Syntax:
{% verbatim %}{{ variable_name|filter_name }}{% endverbatim %}

Filters can be chained, with the output of one filter applied to the next. See example below: 

{% verbatim %}{{ scores_list|random|add:"1"|divisibleby:"2" }}{% endverbatim %}

Filter Description Example usage
add

It is used to add an argument to the value. If that value is a list, then it appends the argument. The argument can be another list. 

{% verbatim %}
# score = 2
{{ score|add:"5" }}
>> score = 7

# [100,90,85]
{{ scores_list|add:"95" }}
>> scores_list = [100,90,85,95]
{% endverbatim %}
addslashes

Adds slashes before quotes. Useful for escaping strings in CSV. 

{% verbatim %}
# some_string = "Where's the Queen's crown?!"
{{ some_string|addslashes }}
>> # some_string = "Where\'s the Queen\'s crown?!"
{% endverbatim %}
capfirst

Capitalizes the first character of the value. If the first character is not a letter, this filter has no effect. 

{% verbatim %}
{{ some_value|capfirst }}
# abc --> Abc
{% endverbatim %}
center

Centers the value in a field of a given width. To use it, you specify a minimum desired length for the given string, and if it's larger than the string, whitespaces will be created around the string (front and back) until the total length of the string matches the desired length and the original string is centered between the whitespaces. 

{% verbatim %}
"{{ value|center:"15" }}"
# "Django" --> "     Django    "
{% endverbatim %}
cut

Removes all values of the argument from the given string.

 
{% verbatim %}
{{ value|cut:" " }}
# "This string has spaces." --> "Thisstringhasspaces."
{% endverbatim %}
date

Formats a date according to the given format. Please refer to this reference chart for a full list of format strings. 

{% verbatim %}
{{ value|date:"D d M Y" }}
# If value is a datetime object, example output:
"Wed 09 Jan 2008"

{{ value|date:"Y-m-d [F]" }}
# "2023-12-31 [January]"

{{ value|date:"SHORT_DATE_FORMAT" }}
# "09/01/2008"
{% endverbatim %}
default

If the value evaluates to False, uses the specified default, else returns the value. 

{% verbatim %}
{{ value|default:"Nothing here." }}
# "" (empty string) --> "Nothing here."
# "abc" --> "abc"
{% endverbatim %}
default_if_none

If, and only if, the value is None, uses the specified default, else returns the value. 

{% verbatim %}
{{ value|default:"Nothing here." }}
# "" (empty string) --> ""
# "abc" --> "abc"
# None --> "Nothing here"
{% endverbatim %}
dictsort

Takes a list of dictionaries and returns that list sorted by the key given in the argument. 

{% verbatim %}
{{ value|dictsort:"name" }}
# if the value is:
[
    {"name": "zed", "age": 19},
    {"name": "amy", "age": 22},
    {"name": "joe", "age": 31},
]
# the output will be:
[
    {"name": "amy", "age": 22},
    {"name": "joe", "age": 31},
    {"name": "zed", "age": 19},
]

# Also works on nested dictionaries
{{ value|dictsort:"author.age" }}
# if the value is:
[
    {"title": "1984", "author": {"name": "George", "age": 45}},
    {"title": "TimeQ", "author": {"name": "Kurt", "age": 75}},
    {"title": "Alice", "author": {"name": "Lewis", "age": 33}},
]
# the output will be:
[
    {"title": "Alice", "author": {"name": "Lewis", "age": 33}},
    {"title": "1984", "author": {"name": "George", "age": 45}},
    {"title": "TimeQ", "author": {"name": "Kurt", "age": 75}},
]

# Works on a list of items, just use an index instead of key
{{ values|dictsort:0 }}
# if the value is:
[
    ("a", "42"),
    ("c", "string"),
    ("b", "foo"),
]
# the output will be:
[
    ("a", "42"),
    ("b", "foo"),
    ("c", "string"),
]
{% endverbatim %}
dictsortreversed

urlize

Converts URLs and email addresses in text into clickable links. This works on linked prefixed with "http://", "https://", or "www" but won't work on links like "goo.gl/aia1t". It also supports domain-only links ending in one of the original top level domains (such as ".com", ".edu", ".gov", ".int", ".mil", ".net", and ".org").

Also converts email addresses into mailto: links. urlize can also take an optional parameter: autoescape. If autoescape is set to True, the link text and URLs will be escaped using Django's escape filter. The default value is set to True.

Links can have trailing punctuation (periods, commas, close-parens) and leading punctuation (opening parens). Links generated by urlize have a rel="nofollow" attribute added to them. This filter should only be applied to plain text. 

{% verbatim %}
{{ value|urlize }}
# if the value is:
"Site: www.djangoproject.com"
# the output will be:
"Site: <a href="http://www.djangoproject.com"
             rel="nofollow">www.djangoproject.com</a>"

{{ value|urlize }}
# if the value is:
"Email: foo@example.com"
# the output will be:
"Email: <a href="mailto:foo@example.com">foo@example.com</a>"
{% endverbatim %}

Custom Template Filters
Setup

If the built-in filters don't match what we need, we can create our own. This requires going through a setup process that involves creating a directory and some py files. Please follow the initial steps below:

  • Create a folder named "templatetags" in the root directory of the app (not the project): 

    ( ... / app_dir / templatetags / )

  • Inside our newly created folder, create a py file named "__init__".

    ( ... / app_dir / templatetags / __init__.py )

  • Then create another py file in the same directory. This file can be named anything, but for our example we will use "custom_filters". 

    ( ... / app_dir / templatetags / custom_filters.py )

The next steps involve editing our custom_filters.py file:

  • Import template from the django module.

    from django import template

  • Instantiate a variable from the template.Library()

    register = template.Library()

  • Create a custom filter:

    • Create a function that accepts "value" as a parameter. A second optional parameter can be added, as needed. Example: 

      def bonkai(value, arg):

    • Create the body of the function so that it performs the desired operation on the value, then returning the result. Example: 

      def bonkai(value, arg):
    return str(value).upper() * int(arg)

  • The last step before the filter can be used is to register the function. For this step we'll use the variable that we used to instantiate the template.Library(). We need to call its filter() function and pass in two arguments: the name of the function cast as a string, and then the function itself. Alternatively, use the variable as a decorator on the function. See below for example: 

    register.filter('bonkai', bonkai)

    # using the template.Library() variable as a decorator
@register.filter('bonkai')
def bonkai(value, arg):
    ...
Example custom filter Python script:
from django import template
register = template.Library()

@register.filter('bonkai')
def bonkai(value, arg):
    return str(value).upper() * int(arg)

# register.filter('bonkai', bonkai) # Can use this or the decorator, not both
Using custom filters

For a webpage to use a custom filter, we need to load the custom filter py file using a template tag. Our example custom filter py script is named "custom_filters", so that's what we need to pass into the template tag, see below: 

{% verbatim %}
{% load custom_filters %}
{% endverbatim %}

To use a filter from our py script, simply call it as you would with a built-in template filter. 

{% verbatim %}
{{ example_value|bonkai:"5" }}
{% endverbatim %}
Example of webpage using built-in and custom filters:
{% verbatim %}
{% extends 'basic_app/base.html' %}
{% load custom_filters %}
{% block title_uno %}
    <title>Index</title>
{% endblock %}
{% block block_one %}
    <h1>Welcome to the Index</h1>
    <h2>text: {{ text }}</h2>
    <h2>text|upper: {{ text|upper }}</h2>
    <h2>text|length: {{ text|length }}</h2>
    <h2>number: {{ number }}</h2>
    <h2>number|add:"23": {{ number|add:"23" }}</h2>
    <h2>text|bonkai:"5": {{ text|bonkai:"5" }}</h2>
{% endblock %}
{% endverbatim %}
