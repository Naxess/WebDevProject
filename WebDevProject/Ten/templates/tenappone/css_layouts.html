CSS - Layouts

"Normal flow" describes the layout of HTML pages by default (i.e. unmodified by layout-changing properties). Elements in the normal flow are displayed in the exact order in which they appear in the code. Elements that appear one after another in this manner are referred to as block elements, in contrast to inline elements, which appear beside one another.

Listed below are methods that can change the layout of elements in CSS:

display property Sets whether an element is treated as a block or inline box and the layout used for its children, such as flow layout, grid, or flex.
  • block: the element generates a block box, generating line breaks both before and after the element when in the normal flow
  • inline: the element generates one or more inline boxes that do not generate line breaks before or after themselves. In normal flow, the next will be on the same line if there is space.
  • flow: the element lays out its contents using flow layout (block-and-inline layout). If its outer display is inline or run-in, and it is participating in a block or inline formatting content, then it generates an inline box, else it generates a block container box.
  • flex: the element behaves like a block-level element and lays out its content according to the flexbox model.
  • grid: the element behaves like a block-level element and lays out its content according to the grid model.
  • none: Turns off the display of an element so that it has no effect on layout. The document is render as though the element doesn't exist. All descendant elements will also have their display turned off.
  • Other values: flow-root, table, ruby, contents
    See full list here.
floats Applying a float value such as left can cause block-level elements to wrap along one side of an element.
position property Allows precise control of the placement of boxes inside other boxes. static positioning is the default in normal flow, but elements can be laid out differently by using other values.
table layout Features designing for styling parts of an HTML table can be used on non-table elements using display: table and associated properties.
multi-column layout Multi-column layout properties can cause the content of a block to lay out in columns.

Introduction to Flexbox Layout

Flexbox is short for Flexible Box Layout, a CSS module designed to make it easy to lay out elements in one dimension - either as a row or column. To use flexbox, set the value of display to flex to the parent element of the elements to be laid out. All of that parent element's direct children then become flex items.

When applied to the parent element, the children will arrange themselves into columns. When they are converted to flex items, they are affected by some initial values that flexbox sets on the flex container. They are displayed in a row because the flex-direction property in the parent element is set torow by default. They appear to stretch in height because of the align-items property of their parent element is set to stretch by default. This causes the items to stretch to the height of the flex container, which is defined by the tallest item. The items are aligned to the start (left) of the container by default.

Using the flex property

In addition to the properties that can be applied to the flex container, there are also properties that can be applied to the flex items. The flex property, when applied to the child item, will grow or shrink the item to fit the space available in its flex container. It has three subcategories:

Subcategory Description
flex-grow specifies how much of the container to use
flex-shrink if larger than container, then shrinks to specified value
flex-basis initial size

The flex property accepts the following values:

Value Description
initial The item is sized according to its width and height properties. It shrinks to its minimum size to fit the cotnainer, but does not grow to absorb any extra free space in the flex container. Equivalent long form: flex: 0 1 auto;
auto The item is zied according to its width and height properties, but grows to absorb any extra free space in the flex container, and shrinks to its minimum size to fit the container. Equivalent long form: flex: 1 1 auto;
none The item is sized according to its width and height properties. It is fully inflexible: it neither shrinks nor grows in relation tot he flex container. Equivalent long form: flex: 0 0 auto;
<'flex-grow'> Defines the flex-grow of the flex item. Negative values are considered invalid. Defaults to 1 when omitted. Initial is 0.
<'flex-shrink'> Defines the flex-shrink of the flex item. Negative values are considered invalid. Defaults to 1 when omitted. Initial is 1.
<'flex-basis'> Defines the flex-basis of the flex item. A preferred size of 0 must have a unit to avoid being interpreted as a flexibility. Defaults to 0 when omitted. Initial is auto.

The flex property accepts between one and three values. The types and number of value(s) supplied will determine which subcategory is supplied:

Value Description
one-value syntax

One of the keywords: auto, initial, and none

Unitless number (i.e. numbers without 'px', 'em', '%', etc..). Value applies to flex-grow. flex-basis is automatically set to 0.

Length unit (e.g. 1cm, 1.2em, 10%, etc..)). Value applies to flex-basis

two-value syntax

Unitless number and length unit. First applies to flex-grow, second applies to flex-basis.

Both values are unitless numbers. First applies to flex-grow, second applies to flex-shrink.

three-value syntax

Unitless number, unitless number, length unit. First applies to flex-grow, second applies to flex-shrink, and third applies to flex-basis.

Introduction to Grid Layout

The grid property is a shorthand property that sets all of the explicit and implicit grid properties in a single declaration.

Using the grid property

As opposed to flexbox's one-dimensional layout, the grid layout is designed for two-dimensional layouts. To enable the grid system, set the value of the display property to grid. Enabling this property means that the grid-template-row and grid-template-columns properties must also be declared. These two properties are used to specify an axis. Three other properties can also be declared: grid-auto-rows, grid-auto-columns, and grid-auto-flow. These three are implicit grid properties.

  • The grid-template-columns property takes a variable number of values. These values can be a fixed length (e.g. 100px), relative length (e.g. 2fr, em), or use the auto keyword. The number of values dictates the number of columns in the grid.
    Examples:
    • grid-template-columns: 1fr 1fr 1fr;
    • grid-template-columns: 100px 100px 100px 100px;
  • The grid-template-rows property takes a variable number of values. These values can be a fixed length (e.g. 100px), relative length (e.g. 2fr, em), or use the auto keyword. The number of values dictates the number of rows in the grid, however, if the number of elements in the grid container exceeds the "capacity" of the grid, they will simply be added as additional rows that follows the column layout.
    Examples:
    • grid-template-rows: 1fr 1fr 1fr;
    • grid-template-rows: 100px 100px 100px 100px;

Grid items have two major properties that are optional: grid-column and grid-row.

  • The grid-column property accepts either one or two grid-line values delimited by a slash ( / ). The first value denotes the start location grid-column-start while the second value denotes the end location grid-column-end or length. Acceptable values include 'auto', 'span', or an integer value.
    Examples:
    • grid-column:2/5;
      grid-row:1;
  • The grid-row property accepts either one or two grid-line values delimited by a slash ( / ). The first value denotes the start location grid-row-start while the second value denotes the end location grid-row-end or length. Acceptable values include 'auto', 'span', or an integer value.

Floats

Floating an element changes the behavior of that element and the block level elements that follow it in normal flow. The floated element is moved to the left or right and removed from normal flow, and the surrounding content floats around it.

The float property has four possible values:

  • left: floats the element to the left
  • right: floats the element to the right
  • none: specifies no floating at all. This is the default
  • inherit: specifies that the value of the float property should be inherited from the element's parent element

Positioning Techniques

Positioning allows movement of an element from where it would normally be in a 'normal flow' to another location. It is not a tool for managing page layouts but for fine-tuning the position of specific elements on a page. There are five types of positioning: static, relative, absolute, fixed, and sticky.

  • Static positioning. This is the default mode. This sets the element in 'normal flow' mode.
  • Relative positioning. Moves an element into position relative to where it would originally be in 'normal flow' mode. It will overlap other elements on the page.
  • Absolute positioning. Moves an element completely out of the page's 'normal flow'. It can then be fixed to a position relative to the dges of its closest positioned ancestor (
  • Fixed positioning. Fixes an element relative to the browser viewport, not another element. This creates a persistent element that stays in the same place on the screen even if the page is scrolled.
  • Sticky positioning. Acts like relative positioning until it hits a defined offset from the viewport, after which it acts according to fixed positioning.

Multi-column Layout

Enabling a multi-column layout is done by declaring the column-count property and setting the value equal to a unitless integer. The number of columns created will reflect this value. The width of each column is equally distributed. Optional properties:

Property Description
column-gap This property changes the size of the gap between columns.
column-rule This property adds a rule between columns (i.e. it draws a line between columns). It accepts up to three values: color, style, and width. It works the same as the border property and even accepts the same values. Some styles: dotted, inset, dashed.

Example usages of column-rule:
  • column-rule: dotted;
  • column-rule: solid 5px;
  • column-rule: solid black;
  • column-rule: solid black;
column-span

This property causes an element to span across all of the columns. It accepts one of two possible values: none or all. The all value causes the element to span across all columns. Content in the normal flow that appears before the element is automatically balanced across all columns before the element appears. The element establishes a new block formatting context.

Example usages:
  • column-span: none;
  • column-span: all;
break-inside Available values: auto, avoid. If the property is set to avoid, it avoids any break (page, column, or region) from being inserted within the principal box. Subvalues: avoid-page, avoid-column, and avoid-region. Simply put, the addition of this property causes the elements to stay in one piece - they will no longer fragment across the columns. Further simplified, when printing, if will force each section to have its own page, rather than split in order to fill a page. See a visual representation here.

Table Layout

This property is enabled by setting the value of the display property to table. This converts the layout of the affected element into that of a table - meaning it will utilize some of the same types of components such as headers, footers, columns, etc.. Each of these components has their own display value. See the list below:

  • table-row: elements affected will behave like a 'tr' element. Example usage: display: table-row;
  • table-cell: elements affected will behave like a 'td' element. Example usage: display: table-cell;
  • table-column: elements affected will behave like a 'col' element. Example usage: display: table-column;
  • table-caption: elements affected will behave like a 'caption' element. Example usage: display: table-caption;
  • table-row-group: elements affected will behave like a 'tbody' element. Example usage: display: table-row-group;
  • table-column-group: elements affected will behave like a 'colgroup' element. Example usage: display: table-column-group;
  • table-header-group: elements affected will behave like a 'thead' element. Example usage: display: table-header-group;
  • table-footer-group: elements affected will behave like a 'tfoot' element. Example usage: display: table-footer-group;
NOTE:

Usage of this feature has lessened due to the inflexible nature of the format, heavy use of markup, difficulty of debugging, and being semantically incorrect. It is now considered a legacy feature.
