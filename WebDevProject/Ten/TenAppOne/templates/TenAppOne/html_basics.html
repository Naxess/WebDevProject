HTML - Basics

Commonly Used Elements and Usages

HTML (HyperText Markup Language) is a markup language that defines the structure of a webpage's content. It consists of a series of elements, which are used to enclose, or wrap, different parts of the content to make it appear or act a certain way.

Webpage Declaration

All HTML documents must be prefaced with a document type declaration: <!DOCTYPE html>. It needs to appear before anything else on the webpage.

The HTML document itself is created when the <html> tags are declared. This tag contains an optional lang attribute.

After the document is created, it must then be supplied with the <head> and <body> tags.

  • The <head> tag can be used as a container for the following elements: <title>, <style>, <link>, <script>, <base>, and <meta>.

    • The <title> tag is required. It defines the title of the webpage.

  • The <body> tag encloses the visible part of the HTML document. There can be only one <body> element on a HTML document at any given time.
Example usages:
<!DOCTYPE html>
<html lang="en-US">
    <head>
        <title>This is the title</title>
    </head>
    <body>
        <h1>This is a header</h1>
    </body>
</html>

Title Tag

The <title> tag defines the title of the webpage.

Example usage:
<title>Webpage Title</title>

Adding a Favicon

A favicon is the icon displayed next to the webpage title. It does not have a tag of its own, but is instead created by adding the rel attribute to link element and setting its value to icon.

Example usage:

Paragraph Tag

The <p> tag creates a paragraph into which we can supply large sections of text. Has an optional title attribute that, when supplied with a text value, will display a tooltip with the given text when the p element is hovered over with a mouse.

Anchor Tag

The <a> tag creates a hyperlink. It requires the use of the href attribute, which needs to be supplied with the link destination.

Example usage:
<a href="https://www.google.com/">This link goes to Google.</a>
This link goes to Google.

Image Tag

The <img> tag allows the webpage to display an image. It requires the use of the src attribute, which needs to be supplied with the location of the image. Other additional attributes are width, height, and alt. The first two attributes are used to specify the dimensions of the image - but something to note is that the image will be stretched or shrink to match. The aspect ratio is not preserved if both are supplied. The alt attribute displays the supplied text only if there is a failure to retrieve the image.

Example usages:

    <img src="catandraccoon.small.png" alt="Uh oh, we couldn't find the image!" width="100">

Form Basics

An HTML form is used to collect user input. To create a form, a form tag needs to be created, inside which the input elements will be placed.

All input elements should specify the name attribute. This attribute is used to identify input fields when it is sent to the server with a GET or POST. The id attribute can also be used alongside or in place of a name field. The name attribute must be unique within a form, while the id must be unique within the whole webpage.

The label tag is optional but provides better aesthetics. The label will appear to the left or right of the input field depending on whether the tag is above or below the input tag in the HTML document.

Examples of regular expressions:
Example expression Description
[A-Za-z]{3} Letters-only (upper and lower case), no numbers. Must have 3 characters.
[0-9]{2,5} Numbers only, no letters. Must have between 2 and 5 characters.
c.t The period ( . ) is a wildcard that accepts any single value. This example accepts any 3 characters (letters and numbers) that start with the letter 'c' and end with the letter 't'. Examples of acceptable values: cat, cot, cut, c@t, c0t, c!t
[a-zA-Z0-9_-]{5} Accepts upper and lowercase letters, numbers, and hypens. Must have 5 characters.
(?=.*\d)(?=.*[A-Z])(?=.*[a-z]).{8} The lookahead expression ( (?=) ) matches the following characters anywhere in the string without any regard to the order.
The wildcard ( . ) when followed by the greedy quantifier ( * ) will match 0 or more times (unlimited, meaning it will find every match in the string instead of only the first).
The digits expression ( \d ) is another way to write ( [0-9] ).

The example expression checks to see that there is at least one number, one lowercase letter, and one uppercase letter within the required 8 characters.
([@#]{1})?([a-z]{8}|[A-Z]{4}) The non-greedy quantifier ( ? ) matches the preceding characters either 0 or 1 times exactly.
A disjunction ( | ) is used between two expressions and specifies that if a string meets the criteria for a match with either the first or second expression of the group, then it will count as a match.

The example expression will match strings that start with @, #, or neither and are either:
8 characters in length and only contain lowercase letters, or
4 characters in length and only contain uppercase letters
