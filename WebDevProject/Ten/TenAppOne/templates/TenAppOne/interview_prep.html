Django

  • What is Django?

    • Django is an open-source web development framework based on the Python language, allowing Python developers to create database-driven websites.

  • Create a simple program that prints "Hello World!"

    • Step 1: Create a new project in the Command Prompt using the following command: 

      django-admin startproject <project_name>

    • Step 2: Change directory to the project's root directory: 

      cd <root_dir_of_project>

    • Step 3: Create and/or open a file named views.py. Add a function that returns an HttpResponse. See code below: 

      from django.shortcuts import HttpResponse

def hello_world_view(request):
    return HttpResponse("Hello World!")

    • Step 4: Create and/or open a file named urls.py. Add a path object to urlpatterns list that contains a reference to the view function we created. See code below: 

      from django.contrib import admin
from django.urls import path
from <project_name> import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.hello_world, name='hello_world')
]

    • Step 5: In the Command Prompt, run the server using the following command: 

      python manage.py runserver

    • The default url is 128.0.0.1. Opening this url in a browser should display a page with the text: "Hello World!"

  • Name some advantages of using Django:

    • High computational and statistical capabilities.

    • Enables developers to create applications and configure frameworks "on-the-go". It also offers support for external libraries and packages. Django focuses on explicit programming, allowing developers to create applications that require changes.

    • Has a REST framework, a python library that helps develop APIs for many applications. Developers can use these APIs to add advanced functionalities to their applications.

    • Comes with Machine Learning capabilities and libraries such as PyTorch, NumPy, etc..

  • Describe Django's architecture.

    • Django follows the Model-View-Template (MVT) architecture, which is based on the Model-View-Controller (MVC) pattern. However, in MTV, our 'views' are referred to as 'controllers' in MVC. In MVC, they have a 'controller' but in MTV, our 'controller' is the framework itself - it is responsible for sending the request to the appropriate view, according to the URL configuration.

    • Django's MVT architecture is divided into 3 major components:

      • Model

        The model handles the database schema for the web applications. It maintains and represents the complete application data into the database. The default relational database used by the model is SQLite, which is generally used in development, but in production can transition to MySQL and PostgreSQL.

      • View

        The view component manages all of the logic of the application that we want to render on the user's browser. In Django, the view acts as a bridge between the models and templates. It's there that it fetches the data from the models and renders that data to the templates.

      • Template

        The template component is the collection os the static parts of the application, such as the HTML, CSS, JavaScript, and image files. The view uses the template as a the base on which the data should be presented, because at the end, the web-application uses the static files to represent the content on the users' browser.

  • Describe the project directory structure in Django.
    • manage.py

      Command-line utility that allows users to interact with their Django project.

    • __init__.py

      An empty file specifying to Python that the current directory is a Python package.

    • settings.py

      Contains the necessary project configurations such as database connections.

    • urls.py

      Contains all of the URLs of the project.

    • wsgi.py

      The application's entry point, used by web servers to serve the project.

  • Describe the purpose of a model in Django.

    • A model is a class that maps to a database table or collection. Model classes contain attributes representing a database field, defined within the models.py file. These models act as the abstraction layer, assisting in structuring and manipulating data. The models are the subclass of the django.db.models.Model class.

    • Example models.py class:

      from django.db import models

class Product(models.Model):
    name = models.CharField(max_length=30)
    price = models.IntegerField()
    description = models.TextField()

    def __str__(self):
        return f'{self.name}'
  • Describe Django's admin interface.

    • The admin interface is the interface already available in Django. It helps developers save time by eliminating the need to create another admin panel. It can be imported from the django.contrib package. It comes with user authentication and other features such as management of various models and CMS, etc..

    • The admin interface has two default models: Groups and Users. These are included in the admin interface after migrating the project and creating a new superuser. Both models fall under the Authentication and Authorization module.

      • Groups

        This table defines the permissions for each group of Users.

      • Users

        This table stores the details of each User.

  • Code reusability is a feature of Django. Describe how Django espouses this concept.

    • Django offers better reusuability than other Python-based web frameworks. It is a collection of various applications, such as login applications, signup, etc.. These applications can be copied from one project to another, requiring only some changes in the settings.py file, thus avoiding the issue of rewriting the entire code from scratch. This is one of the reasons why web applications can be developed quickly in Django.

    • Django is a batteries-included web framework. This means it comes with a variety of build-in common components such as login, signup, session, and authentication features. The code for these pre-written components can imported into our scripts.

    • In the below example, we create a built-in login form instead of creating it manually:

      Example urls.py:

      from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('account/', include('django.contrib.auth.urls')),
]

      Example example.html

      Example settings.py:

      ...
TEMPLATES = [
    {
        ...
        'DIRS': [BASE_DIR / 'templates'],
        ...
    },
]

LOGIN_REDIRECT_URL = '/'
...

      Now, if we open http://127.0.0.1:8000/accounts/login/ in the browser, we should see two text fields ('Username' and 'Password') as well as a button that we created ourselves. The text fields were generated automatically, reducing the amount of manually written code. This is one of the ways that Django implements code reusability.
