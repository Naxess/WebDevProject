Regex - Reference

Basics

Pattern

Description

Example Usage

*

Matches the preceding character 0 or more times.

+

Matches the preceding character 1 or more times.

.

Wildcard. Matches any single character.

x|y

Or. Matches either x or y.

\

Escapes a special character. See here

x

Matches the x character.

xyz

Matches the xyz string.

Character Classes

Pattern

Description

Example Usage

\d

Matches a numeric character.

\D

Matches a non-numeric character.

\s

Matches a single whitespace cahracter (space, tab, form feed, or line feed).

\S

Matches a single non-whitespace character.

\w

Matches any alphanumeric character (including underscores).

\W

Matches any non-word character. "word" characters are defined as a letter, digit, or underscore.

[xyz]

Matches any of the characters within the brackets.

[^xyz]

Matches anything not within the brackets.

[\b]

Matches a backspace.

Assertions

Pattern

Description

Example Usage

^

Matches the beginning of the input.

$

Matches the end of the input.

\b

Matches a word boundary.

\B

Matches a non-word boundary.

?=

Lookahead.

?!

Negative lookahead.

?<=

Lookbehind.

?

Negative lookbehind.

?>

Once-only subexpression.

?()

If-then condition.

?()|

If-then-else condition.

?#

Comment.

Quantifiers

Pattern

Description

Example Usage

{n}

Matches exactly n occurences of the preceding character.

{n,m}

Matches at least n and a maximum of m occurences of the preceding character.

?

Matches 0 or 1.

Special Characters

Pattern

Description

Example Usage

\cX

Matches the control character, X, in a string.

\n

Matches a line feed.

\r

Matches a carriage return.

\t

Matches a tab.

\0

Matches a NULL.

\f

Matches a form feed.

\v

Matches a vertical tab.

\xhh

Matches a character with the code hh (2 hex digits).

\uhhhh

Matches a character with the code hhhh (4 hex digits).

Flags

Pattern

Description

Example Usage

g

Enables global search.

i

Enables case-insensitive search.

m

Enables multi-line search.

y

Enables a "sticky" search starting at the current position in the target string.

Groups

Pattern

Description

Example Usage

(x)

Matches x and remembers the match.

(?:x)

Matches x but does not remember the match.

\n

A back reference to the last substring matching the n parenthetical in the regex.
